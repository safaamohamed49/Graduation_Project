TABLE: financial_accounts (الخزائن والبنوك)
==================================================

COLUMNS:
- id → المعرف الأساسي
- branch_id → الفرع التابع له الحساب المالي
- name → اسم الحساب (مثال: صندوق الفرع الرئيسي / بنك الجمهورية)
- code → رمز الحساب (فريد)
- type → نوع الحساب (cash / bank)
- account_id → الحساب المحاسبي المرتبط (accounts)
- currency → العملة (افتراضي LYD)
- opening_balance → الرصيد الافتتاحي

- notes → ملاحظات
- created_at / updated_at → تواريخ

RELATIONS:
- financial_accounts → belongsTo → branches
- financial_accounts → belongsTo → accounts
- financial_accounts → hasMany → payment_vouchers
- financial_accounts → hasMany → receipt_vouchers

FOREIGN KEYS:
- branch_id → branches.id

PENDING FK:
- account_id → accounts.id

NOTES:
- هذا الجدول يمثل:
  - الصندوق (cash)
  - البنك (bank)

- كل عملية:
  صرف → تخرج من financial_account
  قبض → تدخل إلى financial_account

- لازم كل حساب مالي يكون مربوط بحساب محاسبي:
  مثال:
  صندوق → حساب "الصندوق"
  بنك → حساب "البنك"

- opening_balance يستخدم فقط كبداية
- بعد ذلك يتم حساب الرصيد من القيود

TABLE: inventory_batches (دفعات المخزون)
==================================================

COLUMNS:
- id → المعرف الأساسي
- product_id → المنتج
- warehouse_id → المخزن
- purchase_invoice_id → فاتورة الشراء التي دخلت منها الدفعة
- quantity → الكمية الأصلية
- remaining_quantity → الكمية المتبقية من الدفعة
- base_price → سعر التكلفة الأساسي
- purchase_price → سعر الشراء
- entry_date → تاريخ دخول الدفعة
- expiry_date → تاريخ الانتهاء (اختياري)
- notes → ملاحظات
- created_at / updated_at → تواريخ

RELATIONS:
- inventory_batches → belongsTo → products
- inventory_batches → belongsTo → warehouses
- inventory_batches → belongsTo → purchase_invoices
- inventory_batches → hasMany → order_item_batches

FOREIGN KEYS:
- product_id → products.id
- warehouse_id → warehouses.id
- purchase_invoice_id → purchase_invoices.id

NOTES:
- هذا الجدول يمثل كل دفعة شراء دخلت إلى المخزن
- كل دفعة لها كمية أصلية وكمية متبقية
- عند البيع بنظام FIFO يتم السحب من أقدم دفعة أولاً
- service لا يجب أن ينشئ دفعات مخزون
- remaining_quantity هو الأساس الحقيقي لمعرفة المتبقي من كل دفعة
- البضاعة نفسها قد تكون موجودة في أكثر من مخزن، لذلك warehouse_id مهم جدًا

EXAMPLES:
- المنتج = حديد 6 لينيه
- المخزن = المخزن الرئيسي
- الكمية = 100
- المتبقي = 65
- base_price = 75
- purchase_price = 80

SYSTEM ROLE:
- أساس إدارة المخزون
- أساس حساب التكلفة
- أساس FIFO
- يحدد الرصيد لكل منتج داخل كل مخزن

TABLE: order_item_batches (ربط بنود البيع بالدفعات)
==================================================

COLUMNS:
- id → المعرف الأساسي
- order_item_id → بند البيع
- batch_id → دفعة المخزون التي تم السحب منها
- quantity_used → الكمية المسحوبة من هذه الدفعة
- unit_cost → تكلفة الوحدة وقت السحب
- total_cost → إجمالي التكلفة لهذا الجزء المسحوب
- created_at / updated_at → تواريخ

RELATIONS:
- order_item_batches → belongsTo → order_items
- order_item_batches → belongsTo → inventory_batches

FOREIGN KEYS:
- order_item_id → order_items.id
- batch_id → inventory_batches.id

NOTES:
- هذا الجدول لا يمثل البيع نفسه
- هذا الجدول يمثل من أي دفعات تم السحب وقت البيع
- إذا تم بيع 10 وحدات من منتج، ممكن تسحب:
  - 6 من دفعة أولى
  - 4 من دفعة ثانية
- لذلك قد يكون لبند بيع واحد أكثر من سجل في هذا الجدول
- هذا الجدول أساسي جدًا في:
  - FIFO
  - حساب التكلفة
  - الإلغاء
  - المرتجعات

EXAMPLES:
- order_item_id = 15
- batch_id = 3
- quantity_used = 6
- unit_cost = 75
- total_cost = 450

SYSTEM ROLE:
- يربط المبيعات بالمخزون
- يحدد تكلفة كل بند بيع بدقة
- يسمح بإرجاع الكمية لنفس الدفعات عند الإلغاء أو المرتجع
